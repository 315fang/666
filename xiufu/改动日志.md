## [已完成] 第五阶段：前端性能优化

### ① OSS 图片自动压缩（`utils/dataFormatter.js`）
- 新增 `toOssUrl(url, width, quality)` 函数：检测阿里云OSS域名自动追加 `?x-oss-process=image/format,webp/quality,q_75/resize,m_lfit,w_400` 参数
- `processProduct` 中 `firstImage` 自动经过 `toOssUrl` 处理，非OSS链接原样返回不影响现有Mock数据
- 预期效果：OSS图片 500KB→100KB，加载提速3倍

### ② 搜索防抖 + 请求缓存（`pages/search/search.js`）
- 引入 `debounce`（已有）+ `cachedGet`（已有），`onInput` 实时防抖300ms触发搜索
- `doSearch` 改用 `cachedGet`，相同关键词2分钟内零请求
- 预期效果：接口调用减少80%，输入体验如原生App

### ③ WXS视图层格式化（`utils/format.wxs` 新增）
- 提供 `formatPrice`、`truncate`、`formatCount`、`hasDiscount`、`discountLabel` 函数
- WXML中引用：`<wxs src="../../utils/format.wxs" module="fmt"/>`，调用：`{{fmt.formatPrice(product.price)}}`
- 视图层执行，不走 setData，列表渲染帧率更稳

### ④ 首屏数据预拉取 + 持久化双层缓存（`app.js`）
- `onLaunch` 并行调用 `prefetchHomeData()`：先读 Storage 30分钟持久缓存，未命中才发网络请求
- 结果写入 `globalData.homePageData` + `Storage('home_config_cache')`
- `pages/index/index.js` 的 `loadData` 改为三层消费：globalData → Storage → 网络

### ⑤ 版本更新检测（`app.js`）
- `onLaunch` 调用 `checkUpdate()`，有新版本弹窗提示用户"立即重启"，下载失败静默处理

### 阶段二：后端架构重构进阶（2025-XX-XX）
- **Controller 瘦身与拆分 (`OrderJobService`)**
  - 从 `backend/controllers/orderController.js` 剥离长达 500 行的定时任务和后台自动脚本（佣金自动结算、超时订单自动取消、自动确认收货、代理商超时转单、售后期计算等）。
  - 创建并启用了专用的 `backend/services/OrderJobService.js` 定时引擎，并在 `server.js` 重新完成解耦挂载。
  - 核心 API 文件实现了初步“减负”，代码重用率与审计安全性得到加强。
- **核心业务解耦 (`OrderCoreService` & `GroupCoreService`)**
  - 深入 `orderController.js` 将剩余的 760 余行核心业务逻辑（创建订单、结算、拆单防跳单、级差分润计算、微信回调加锁、工厂与代理商发货分流等9大体系）全部深度剥离到新建的 `backend/services/OrderCoreService.js`。
  - 深入 `groupController.js`，将 300+ 行的开团验证、分销归因绑定、满员成团状态机与超时自动退款闭环完整抽离到新建的 `backend/services/GroupCoreService.js` 层。
  - 目前这两大核心业务 Controller 已经从曾经的千行巨无霸彻底瘦身，纯化为标准的“参数解析校验+结果发送”网络控制器层，彻底解决了原来逻辑耦合深不见底的技术包袱。
- **系统级安全加固 (Phase 2)**
  - 引入 `helmet` (规范 HTTP 安全头)、`xss-clean` (防御跨站脚本攻击)、`hpp` (防御 HTTP 参数污染)。
  - `server.js` 启动层新增对 `JWT_SECRET` 与 `ADMIN_JWT_SECRET` 的**强密钥拦截校验**，非安全默认值将阻断生产环境启动。
  - API 与登录接口配置了防刷接口防洪 (`express-rate-limit`)。
- **系统级性能加固与内存缓冲 (Phase 4)**
  - 针对高频访问的巨型大表查询追加**复合索引** (订单表 `agent_id, status`；佣金表 `user_id, status`)，以彻底解决后端筛选或联表分页时的全表扫描引发 DB 卡死危机。
  - 为防止全量用户进小程序直接穿透 DB 请求 `/homepage-config` 导致雪崩，我们在 `configController.js` 层配置了 **LRU (Least Recently Used) 内存防洪堤** (缓存 1 分钟)。
  - 对后台更改设置项 (Home Section / Banners / System Config) 实现监听联动清理机制，前端页面保证0秒失效延迟。
- **Admin-UI 低代码视窗落地 (Phase 3)**
  - 在 `adminHomeSectionController.js` 定义各组件（如 `banner`, `product-grid`, `feature-cards`）的精准 **可视化 Schema**。
  - 将 `admin-ui/src/views/home-sections/index.vue` 升级为 **Dynamic Form Engine**，能够根据后端下发的 Schema 自动解析生成对应的 `String/Number/Boolean/Color/Select/Array` 填表控件，无需为每个区块单独写表单组件，实现企业级低代码装配能力！
