# 开发手册（全栈 · 接口 · 上线）

本文档面向日常开发和联调，强调当前仓库里真实存在的系统与操作方式。

业务范围先看 [`architecture/项目业务总览.md`](./architecture/项目业务总览.md)，技术边界先看 [`architecture/overview.md`](./architecture/overview.md)。

## 1. 当前仓库结构

| 目录 | 说明 |
|------|------|
| `backend/` | 当前现网主后端，承载 `/api`、`/api/n`、`/api/agent`、`/api/portal`、`/admin/api` |
| `admin-ui/` | 当前现网管理后台前端 |
| `miniprogram/` | 当前现网微信小程序 |
| `cloud-mp/` | CloudBase 迁移线，不是当前现网主线 |
| `docs/` | 当前文档主入口 |

说明：

- 当前仓库里没有可作为主线维护对象的 `agent-portal/`
- 后端仍保留门户 API，但不应据此假设门户前端仍在本仓库

## 2. 本地运行

### 2.1 后端

```bash
cd backend
npm install
cp .env.example .env
npm run dev
```

默认端口 `3001`，常用入口：

- `GET /health`
- `GET /api/docs`

### 2.2 管理后台

```bash
cd admin-ui
npm install
npm run dev
```

本地通常通过 `/admin` 打开页面，通过 `/admin/api` 访问接口。

### 2.3 微信小程序

- 用微信开发者工具打开 `miniprogram/`
- 按 `miniprogram/config/env.js` 配置 API 环境

## 3. 代码入口怎么找

| 主题 | 优先代码入口 |
|------|--------------|
| 下单 / 支付 / 回调 | `backend/services/OrderCoreService.js`、`backend/services/OrderCreationService.js`、`backend/services/OrderPaymentService.js` |
| 佣金 / 分销 / 提现 | `backend/services/CommissionService.js`、`backend/routes/commissions.js`、`backend/routes/wallet.js` |
| 标准代理 / 货款钱包 | `backend/routes/agent.js`、`backend/services/AgentService.js`、`backend/services/AgentWalletService.js` |
| N 路径 | `backend/routes/n-system.js`、`backend/services/NSystemService.js`、`backend/services/UpgradeMemberService.js` |
| 内容 / Banner / 页面编排 | `backend/routes/config.js`、`backend/routes/page-content.js`、`backend/routes/boards.js` |
| 管理端权限 | `backend/config/adminPermissionCatalog.js`、`admin-ui/src/router/index.js` |

## 4. 当前接口面

### 4.1 URL 前缀

| 客户端 / 场景 | Base |
|--------------|------|
| 小程序与公开接口 | `/api` |
| 标准代理工作台与货款 | `/api/agent` |
| N 路径 | `/api/n` |
| 门户后端接口 | `/api/portal` |
| 管理后台 | `/admin/api` |

### 4.2 后端挂载重点

`backend/app.js` 当前挂载了以下核心域：

- 用户、商品、分类、购物车、订单、退款、地址、内容、配置
- 分销、佣金、钱包、代理工作台、N 路径
- 拼团、砍价、抽奖、优惠券、积分、活动
- 自提门店、物流、开屏、推荐位、页面内容
- 门户鉴权与门户业务接口
- 管理端聚合接口

### 4.3 门户说明

当前真实状态是：

- 后端仍保留 `/api/portal/auth/*` 和 `/api/portal/*`
- 小程序仍保留门户口令申请链路
- 但门户前端源码不在当前仓库，不应当作本地必跑项目

## 5. 上线发布检查

1. 确认数据库迁移已按当前环境执行
2. 生产 `.env` 中 JWT、管理员 JWT、数据库、微信支付、CORS 均使用真实安全配置
3. 确认微信支付回调地址为 `/api/wechat/pay/notify`
4. 确认 `admin-ui/dist` 已构建并与后端一起部署
5. 确认生产关闭 `ENABLE_DEBUG_ROUTES`、`ENABLE_TEST_ROUTES`
6. 业务验收至少覆盖下单、支付、发货、自提、退款、提现、活动展示、内容位展示

## 6. 冒烟建议

- 小程序登录、首页、分类、详情、购物车
- 下单、拉起支付、支付回调、订单状态流转
- 优惠券、积分、活动页、自提、物流
- 分销中心、佣金列表、提现申请
- 后台商品保存、订单处理、内容位配置、权限校验

## 7. 文档维护

- 改业务边界：先改 [`architecture/项目业务总览.md`](./architecture/项目业务总览.md)
- 改技术事实：先改 [`architecture/overview.md`](./architecture/overview.md)
- 改接口：同步 Swagger 和前端 `api` 封装
- 改专题规则：同步 [`业务规则.md`](./业务规则.md) 或对应 `internals/` 文档
