# 必要文档清单与维护策略

## 1. 目标

这份文档回答一个问题：

**这个项目为了长期维护，最少需要哪些文档。**

这里不是列“想写什么”，而是列“没有它就会反复失真、反复返工”的文档集合。

## 2. 必要文档的判断标准

只要满足以下任一条件，就属于必要文档：

- 新人或 AI 不看就无法快速理解系统
- 业务规则复杂且容易误改
- 多模块耦合明显
- 改动风险高
- 迁移过程中容易出现双真相
- 同一个问题已经反复出现

## 3. 最小必要文档集合

## 6. 维护策略

### 6.1 谁应该更新

谁改动了高风险域，谁就负责更新对应文档。

### 6.2 什么时候更新

以下情况必须同步更新文档：

- 订单链路变化
- 支付口径变化
- 佣金规则变化
- 权限模型变化
- 字段口径变化
- CloudBase 迁移阶段变化

### 6.3 如何避免文档再次失真

1. 文档必须引用真实代码和真实目录。
2. 文档必须明确“现网真相”和“迁移目标”是否是同一套。
3. 文档必须能指导下一步行动，而不是停留在描述层。
4. 文档治理规则以 [docs/rules/document-governance-rules.md](C:\Users\21963\WeChatProjects\zz\docs\rules\document-governance-rules.md) 为准。
