# 核心业务规则

## 订单

1. controller 只负责 request/response，核心业务进入 service。
2. 下单、预支付、取消、确认收货、发货属于关键链路。
3. 拆单后子订单独立履约，但详情展示要能关联主链路。

## 支付

1. 小程序支付以预支付 + 微信回调为主路径。
2. 待付款允许补查微信支付状态，缓解回调缺失导致的状态滞后。
3. 钱包支付偏好不能绕过订单状态和金额校验。

## 发货与收货

1. 发货区分平台发货与代理发货。
2. 确认收货后才进入售后期和后续佣金链路。
3. 自提与快递的前后端能力和文案必须一致。

## 权限

1. 权限键唯一真相源是后端权限目录。
2. `super_admin` 是角色，不是普通权限键。
3. alias 仅用于旧数据兼容，新代码禁止继续写入。

## 文档

1. README、审计报告、总方案、任务清单必须同步。
2. 失真历史文档不再作为主入口。
3. 文档与代码冲突时，以当前代码和审计结论为准。
