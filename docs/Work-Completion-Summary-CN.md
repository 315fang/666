### 第2部分: 工具函数（utils/dataFormatter.js）

**我新增了3个函数**:

#### 函数1: formatGrowth() - 计算增长率

```javascript
// 使用示例
const growth = formatGrowth(2847.50, 2500.00);
// 返回: { value: '+13.9', trend: 'up', percentage: 13.9 }

// 如果下跌
const decline = formatGrowth(2300.00, 2500.00);
// 返回: { value: '-8.0', trend: 'down', percentage: -8.0 }
```

**为什么需要这个**:
- 你需要显示"本月收益 +28.5%"这样的增长率
- 需要知道是上涨（绿色↑）还是下跌（红色↓）
- 这个函数帮你自动计算和判断

#### 函数2: normalizeTrendData() - 标准化趋势数据

```javascript
// 使用示例
const rawData = [500, 800, 1200, 1000, 1500];
const normalized = normalizeTrendData(rawData);
// 返回: [33, 53, 80, 67, 100]  ← 转换成百分比

// 为什么: 柱状图需要百分比高度，不是原始金额
```

**为什么需要这个**:
- 趋势图的柱状图高度应该是百分比（0-100%）
- 不能直接用金额（500, 800, 1200）作为高度
- 这个函数把原始数据转换成百分比

#### 函数3: formatLargeNumber() - 格式化大数字

```javascript
// 使用示例
formatLargeNumber(156);      // → "156"
formatLargeNumber(1500);     // → "1.5k"
formatLargeNumber(15000);    // → "1.5万"
```

**为什么需要这个**:
- 如果团队有15000人，显示"15000"太长
- 显示"1.5万"更简洁易读
- 符合中文用户习惯

**这份文档讲什么**:

**1. 为什么需要插画？**
```
问题: 纯图标和文字太冰冷
示例: 空购物车显示 "购物车是空的" ❌
改进: 小人推着金币购物车 ✅

效果对比:
冰冷版: "购物车是空的" → 用户感觉被拒绝
温暖版: 小人+购物车+金币 → 用户感觉"我可以去赚钱"
```

**2. 我的4大设计原则**

```
原则1: 功能先于美观 Function Over Form
- 插画不是装饰品，是信息传达工具
- 错误: 花园插画（好看但不知道是购物车）
- 正确: 购物车+小人（一眼看懂）

原则2: 简约但有细节 Simple But Detailed
- 太简单: 只有线条（太抽象）
- 太复杂: 像照片（信息过载）
- 刚刚好: 扁平化+线性风格（识别度高）

原则3: 叙事性 Storytelling
- 每个插画都在讲一个小故事
- 空购物车: "空的 → 可惜 → 来填满它！"
- 首单成功: "挑战 → 努力 → 成功 → 庆祝"

原则4: 一致性 Consistency
- 所有插画使用相同的风格
- 使用你的品牌色（深空灰+香槟金）
- 统一的人物形象（3头身小人）
```
