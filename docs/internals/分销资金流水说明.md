# 分销资金流水说明

## 1. 为什么需要这份文档

分销不是单一“佣金数字”问题，而是几类资金同时存在：

- 佣金
- 钱包余额
- 提现
- 退款冲减
- 冻结期与结算期

如果没有这份文档，后续很容易把“看到的钱”和“真正可结算的钱”混在一起。

## 2. 当前涉及的核心对象

从现有后端模型与控制器看，分销资金链至少涉及：

- `CommissionLog`
- `CommissionSettlement`
- `Withdrawal`
- `AgentWalletLog`
- 订单状态与退款状态

## 3. 佣金流水主状态

当前 `CommissionLog` 注释中能看到的核心状态包括：

- `frozen`
- `pending_approval`
- `approved`
- `settled`
- `cancelled`

这组状态表达的是：

- 佣金是否已经产生
- 是否进入审批
- 是否真正结算
- 是否因为退款或规则变化被取消

## 4. 订单状态与佣金状态的关系

当前业务里，订单不是一支付就能最终结算佣金。

中间至少还隔着：

- 发货
- 确认收货
- 售后期
- 退款截止

从现有代码注释看，确认收货后会写入：

- `completed_at`
- `refund_deadline`
- 或与之等价的售后期截止时间

这一步很关键，因为它决定：

- 冻结佣金何时可推进
- 退款是否仍会冲减佣金

## 5. 当前资金流的正确理解

### 5.1 用户付款

用户付款后：

- 订单进入履约链
- 不代表佣金立即可提现

### 5.2 订单完成

订单完成后：

- 佣金通常进入冻结或待审批阶段
- 还要看售后期是否结束

### 5.3 售后期结束

售后期结束后：

- 冻结佣金才有资格进入结算

### 5.4 申请提现

提现处理的是：

- 已可提现余额

不是：

- 总佣金
- 冻结佣金
- 尚未审核佣金

## 6. 退款对资金链的影响

退款会影响至少三层：

- 订单收入
- 佣金流水
- 钱包或提现口径

因此退款不是单纯改一张 `refunds` 表，而是会联动冲减或取消佣金资格。

这也是为什么：

- 订单
- 售后
- 佣金
- 提现

必须被视为同一条高风险链。

## 7. 当前阶段的长期建议

### 7.1 不再扩展复杂代理结算语义

当前优先保留最小闭环：

- 佣金中心
- 团队
- 提现
- 佣金流水

不优先继续扩展：

- 更复杂的代理层级结算
- 多口径看板并存
- 页面层重复计算资金结果

### 7.2 页面只展示，不自己算真相

后台和小程序页面只能展示后端给出的资金结果，不能在页面层自己拼凑：

- 可提现余额
- 冻结金额
- 预计收益

### 7.3 资金状态更新要有审计意识

任何会影响资金口径的改动，都应同时检查：

- 订单状态推进
- 售后期截止
- 佣金日志状态
- 提现状态

## 9. 当前必须遵守的规则

1. 不把“总收益”当“可提现余额”。
2. 不把已支付订单直接当成可结算佣金。
3. 退款影响佣金时，必须走状态链和流水链，不允许只改页面展示。
4. 分销资金相关代码变更，必须同步更新本文件和分销总览文档。
