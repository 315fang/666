这份文档只回答一个问题：

**小程序主要页面在依赖哪些核心接口。**

它不是完整 API 文档，而是帮助后续做：

- 页面改造
- CloudBase 迁移
- 接口收口
- 风险定位

## 3. Cloud Function 映射

当前 CloudBase 小程序路由最终落到这些云函数：

- `login`
- `user`
- `products`
- `cart`
- `order`
- `payment`
- `config`
- `distribution`

说明：

- 页面级改动如果涉及数据读写，必须先确定落在哪个云函数
- 不要绕过 `utils/request.js` 直接在页面里扩散新接口口径

## 4. 当前最需要优先收口的页面域

根据兼容审计，优先级最高的是：

1. `cart`
2. `order`
3. `distribution`
4. `index`
5. 内容装修相关展示页

原因：

- 旧字段依赖最多
- 状态兼容最多
- 直接影响交易闭环或首页展示

## 5. 使用建议

后续任何页面改造，先回答三个问题：

1. 这个页面属于哪个业务域
2. 它当前走哪个云函数
3. 它是否仍依赖旧字段兼容

不先回答这三个问题，就容易继续把石山代码越堆越大。
