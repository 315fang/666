# 小程序分层约定

## 目标

把页面主文件收成“页面壳”，把数据装配和动作逻辑拆到同目录 helper 或通用 utils。

## 约定

### 页面主文件

保留：

- 生命周期
- 页面 data
- 少量事件转发
- 组件引用与轻量导航

不再长期堆放：

- 大段接口编排
- 多分支支付/地图/倒计时逻辑
- 可复用的分组、文案映射、金额计算

### 页面 helper

适合放页面目录下：

- `xxxLoader.js`
- `xxxData.js`
- `xxxActions.js`
- `xxxPayment.js`

适用场景：

- 页面内数据装配
- 页面内动作逻辑
- 地图、支付、账户、倒计时等单页子域

### 通用 utils

跨页面复用逻辑进入 `miniprogram/utils/`：

- request
- errorHandler
- navigator
- userProfile
- miniProgramConfig

## 当前已按此约定拆分的页面

- 用户页
- 分类页
- 订单确认页
- 首页
- 商品详情页
- 订单详情页
- 活动页
- 自提站地图页
- 代理钱包页

## 约束

1. 不再新增 15KB 以上且混合多类职责的页面主文件
2. 页面主文件不再保留大量历史备用逻辑
3. 新逻辑优先考虑 helper，再考虑直接塞回页面
