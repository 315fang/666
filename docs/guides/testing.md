# 测试说明

## 当前状态

后端测试命令当前以 Jest 为准。

已确认可执行：

```powershell
cd backend
npm test
```

## 统一规则

1. `backend/__tests__/` 下测试统一使用 Jest。
2. 不允许再把 `node:test` 和 Jest 混用。
3. 后续新增测试全部沿用 Jest 风格。
4. 路由权限测试优先使用 `supertest` 覆盖真实中间件挂载路径。

## 当前测试分层

### controller

用于验证 request/response 形态、XML / JSON 输出、错误映射。

### middleware

用于验证权限放行、拒绝、历史别名归一化。

### service

用于验证订单号、缓存、金额、佣金等服务层逻辑。

对已经收口为协调器的服务，例如 `OrderCoreService`，优先验证公开方法是否稳定委托到正确子服务，并保持错误传播语义不变。

## 当前覆盖范围

当前已接入的测试以 `services/`、`utils/`、部分 `controllers/` 为主。

重点说明：

- 现阶段测试数量仍然有限
- 但至少 `npm test` 已恢复为真实可执行、真实可判断结果
- 订单主链路当前已覆盖 `createOrder / prepayOrder / cancelOrder / confirmOrder` 的协调层契约
- 履约规则当前已覆盖售后期截止时间与确认收货事务边界
- 佣金规则当前已覆盖全局比例、商品固定佣金优先级、利润池不足时的分配行为
- 支付回调当前已覆盖空回调、签名失败、重复通知、金额不一致、订单不存在、正常入账分支

## 后续优先链路

下一批优先补的测试链路：

1. 管理员权限
2. 订单创建
3. 支付状态同步
4. 发货与确认收货
5. 退款与售后
