# 开发手册（全栈 · 接口 · 上线）

面向日常开发：**本地运行、接口对照、发版检查、支付证书、配置驱动摘要**。  
业务术语与佣金口径见 **[业务规则.md](./业务规则.md)**；后端安装与部署细节见 **[backend/README.md](../backend/README.md)**；管理端与门户见 **[前端.md](./前端.md)**。  
机器可读接口清单：启动后端后访问 **`/api/docs`（Swagger）**。

---

## 1. 仓库结构

| 目录 | 说明 |
|------|------|
| `backend/` | Express + Sequelize + MySQL → `/api`、`/admin/api`、`/api/portal` |
| `miniprogram/` | 微信小程序，`baseUrl` 一般为 `https://域名/api` |
| `admin-ui/` | Vue3 管理后台 |
| `agent-portal/` | Nuxt 代理门户 |
| `docs/` | 本手册、`业务规则.md`、`前端.md`、可再生成的路径对照快照（勿在仓库外镜像目录当主线改代码） |

---

## 2. 本地运行（摘要）

**后端**：`cd backend && npm i && cp .env.example .env && npm run dev` → 默认 `:3001`。  
**小程序**：微信开发者工具打开 `miniprogram/`，配 `miniprogram/config/env.js`。  
**管理后台**：`cd admin-ui && npm i && npm run dev` → `/admin/`。  
**门户**：`cd agent-portal && npm i && cp .env.example .env && npm run dev`。

迁移：执行 `backend/migrations/` 下未跑过的 SQL（见下文「上线检查」）。

## 4. 前后端接口对照

### 4.1 URL 前缀

| 客户端 | Base |
|--------|------|
| 小程序 | `…/api` + 代码里的 `/login` 等 |
| 管理后台 | `/admin/api` |
| 门户 | `NUXT_PUBLIC_API_BASE` 为域名根 + 路径 `/api/portal/...` |

### 4.2 后端挂载（`app.js`）

- **C 端 `/api`**：`auth`、`products`、`orders`（含微信 notify）、`addresses`、`distribution`、`users`、`categories`、`cart`、`materials`、`content`、`wallet`、`refunds`、`dealer`、`agent`、`upgrade`、`commissions`、`config`（`configs`、`banners`、`homepage-config`、`mini-program-config`、`rules`…）、`themes`、`points`、`group`、`activity`、`lottery`、`coupons`、`slash`、`pickup`、`stations`、`logistics`、`splash`、`boards`、`page-content`  
- **管理端**：`/admin/api/*`（`routes/admin/index.js`）  
- **门户**：`/api/portal/auth/*`、`/api/portal/*`  
- **其它**：`GET /health`、`/api/docs`、调试路由（受环境变量开关控制）

### 4.3 小程序常用路径（相对 `/api`）

登录与用户：`POST /login`，`GET/PUT /user/profile`，`POST /user/upload`，`GET /user/member-tier-meta`  
通知：`GET /notifications`，`PUT /notifications/:id/read`  
偏好：`GET /user/preferences`，`POST /user/preferences/submit`  
配置首页：`GET /configs`，`GET /mini-program-config`，`GET /homepage-config`，`GET /page-content/home`，`GET /themes/active`，`GET /splash/active`  
商品：`GET /banners`，`GET /products`，`GET /products/:id`，`GET /products/:id/reviews`  
购物袋（接口仍为 cart）：`GET/POST /cart`，`PUT /cart/:id`，`DELETE /cart/:id`  
地址：`GET/POST /addresses`，`GET/PUT /addresses/:id`，`DELETE /addresses/:id`，`POST /addresses/:id/default`  
订单：`GET/POST /orders`，`GET /orders/:id`，`POST /orders/:id/prepay|confirm|cancel|review`（支付结果由微信调 `POST /api/wechat/pay/notify`）  
售后：`GET/POST /refunds`  
分销钱包：`GET /distribution/overview|stats|team`，`GET /stats/distribution`，`GET /wallet/info`，`GET /wallet/commissions`，`POST /wallet/withdraw`，`GET /wallet/withdrawals`  
代理货款：`GET /agent/workbench`，`GET /agent/wallet`，`GET /agent/wallet/logs`，`GET /agent/wallet/recharge-config`，`POST /agent/wallet/prepay`  
佣金预览：`GET /commissions/preview`（可能与实付不完全一致）  
积分：`GET /points/account|tasks|logs`，`GET /points/sign-in/status`，`POST /points/sign-in`  
拼团 / 活动 / 砍价 / 抽奖 / 优惠券 / 自提 / 物流：同前版 `API_SURFACE_MAP`（`group/*`、`activity/*`、`slash/*`、`lottery/*`、`coupons/*`、`stations/pickup-options`、`logistics/order/:id` 等）

**管理后台**：`admin-ui/src/api/index.js` 与 `routes/admin/index.js` 一一对应；报错先查 RBAC `checkPermission`。

**门户**：见 [前端.md](./前端.md) 表格。

### 4.4 小程序未接但后端已有（产品取舍）

| 接口 | 说明 |
|------|------|
| `/api/dealer/*` | 小程序无调用；要「经销商」需补页面 |
| `/api/upgrade/*` | 小程序无调用；付费升级需对接 |
| `POST /api/bind-parent` | 预留，无现成 C 端入口 |
| `PUT /api/cart/select/batch` | 服务端批量选中；小程序未用 |
| `GET /api/quick-entries`、`GET /api/home-sections` | 常被 `homepage-config` 聚合替代 |

### 4.5 后端有但某端几乎不用

`GET /api/portal/dashboard/summary`（门户未调）；`POST /api/orders/:id/pay`（仅测试开关）；`/admin/api/test/*`、`/api/debug/*`；管理端 CRUD 不被小程序直接调（正常）。

### 4.6 维护约定

改路由 → 同步 Swagger + 更新本节表述 + 前端 `api` 封装。

## 5. 上线发布检查

1. **迁移**：至少核对已执行 `20260321_activity_spot_stock.sql`、`20260321_order_completion_processed.sql` 及历史 `202602*.sql`、`009_*.sql`、`010_*.sql` 等；未执行则相关功能可能报错。  
2. **环境**：从 `backend/.env.example` 配生产 `.env`；JWT/ADMIN_JWT、DB、微信、**支付 V3** 不用默认值；`CORS_ORIGINS` 不用 `*`；对象存储按 `STORAGE_PROVIDER` 配置。  
3. **微信支付**：统一下单 `utils/wechat.js`；回调 **`https://<API域名>/api/wechat/pay/notify`**；商户平台配证书、V3 密钥、回调 URL、与小程序 AppID 绑定；小程序配置合法域名与 HTTPS。  
4. **业务验收**：自提、限时专享、活动链、优惠券指定范围等在后台与小程序对照。  
5. **运维**：Node 版本、PM2/日志、定时任务（`OrderJobService` 等）在生产已启用；备份与回滚策略。

## 7. 生产运维要点（精简）

- 已实现 **微信 V3 统一下单 + JSON 回调**，勿按旧文档「仅模拟支付」理解。  
- 生产 **关闭** `ENABLE_DEBUG_ROUTES`、`ENABLE_TEST_ROUTES` 等。  
- 随机密钥：`node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`  
- **多实例**：上传走 OSS/COS；定时任务可设 `TASK_LOCK_BACKEND=mysql`（`utils/taskLock.js`，注意连接池）。  
- 增强项：日志轮转、DB 备份、可选 Sentry、逐步补自动化测试。

## 8. 工程备忘（安全 · 佣金）

1. 支付回调在全局限流中排除，靠签名与金额校验。  
2. 管理员 Token 黑名单：`ADMIN_TOKEN_BLACKLIST_STORE` = memory / redis / mysql（`npm run migrate:phase9`）。  
3. `GET /api/orders/:id`：买家或该单 `agent_id` 可查。  
4. `orders.completion_processed`：完成统计幂等。  
5. 提成基数以支付后 **`Order.actual_price`** 为准。  
6. `users.participate_distribution`：后台 `PUT /admin/api/users/:id/commerce`。

## 9. 微信支付 V3 平台证书（自动更新）

**目标**：启动后拉取平台证书、写入 PEM、按周期刷新（默认 24h）。

**必填 `.env`**：`WECHAT_APPID`、`WECHAT_MCH_ID`、`WECHAT_PAY_SERIAL_NO`、`WECHAT_PAY_API_V3_KEY`（32 位）、`WECHAT_PAY_PRIVATE_KEY_PATH`、`WECHAT_PAY_NOTIFY_URL`。  
**建议**：`WECHAT_PAY_PLATFORM_CERT_PATH=certs/wechatpay_platform_cert.pem`、`WECHAT_PAY_CERT_AUTO_PERSIST=true`、`WECHAT_PAY_CERT_REFRESH_INTERVAL_HOURS=24`。

**行为**：`server.js` 预热证书；定时刷新；验签优先内存证书，失败回退本地 PEM。  
**排查**：`GET /admin/api/debug/cron-status`（含 certStatus）；查商户号/序列号/私钥、APIv3 长度、出网、`WECHAT_PAY_PLATFORM_CERT_PATH` 写权限。

## 10. 配置驱动首页（摘要）

**思想**：可变内容（轮播、金刚区、首页区块、部分全局参数）走库表，小程序只渲染。

**主表**：`app_configs`（键值配置）、`quick_entries`（快捷入口）、`home_sections`（首页区块）。  

**推荐接口**：`GET /api/homepage-config`（一次拉齐）；另有 `GET /api/configs`、`GET /api/quick-entries`、`GET /api/home-sections`、`GET /api/banners?position=`。  

**集成**：先取 `homepage-config`，再按配置拉商品列表等；入口点击按 `link_type` / `link_value` 路由。

**小程序常见图片类配置键**（运维对照，多来自 `app_configs` 等）：`home_poster_url`、`guide_bg`、`cocreation_bg`。

## 11. 仓库冗杂治理与路径对照脚本

- **主线目录**：仅在 `backend/`、`admin-ui/`、`miniprogram/`、`agent-portal/`、`docs/` 中迭代；勿在本地再建整树镜像目录改代码（历史快照已从工作区移除的可再建于仓库外）。
- **admin-ui API**：`admin-ui/src/api/modules/*` 按业务域拆分，`admin-ui/src/api/index.js` 统一 `export *`，页面仍从 `@/api` 引用。
- **管理端路径粗对照**：在仓库根执行 `node scripts/audit-admin-api-alignment.mjs`，覆盖写入 [管理端前后端路径对照快照.md](./管理端前后端路径对照快照.md)。脚本未解析 `router.use` 挂载前缀时，报告中「仅后端」可能出现 `/:id` 等相对片段，需结合 [backend/routes/admin/index.js](../backend/routes/admin/index.js) 人工判断；权威契约仍以 Swagger `/api/docs` 为准。
- **根目录说明**：执行记录与约定摘要见 [代码冗杂治理说明.md](../代码冗杂治理说明.md)。
