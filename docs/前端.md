# 前端子项目（管理后台 · 微信小程序）

本文档只描述当前仓库里真实存在并应继续维护的前端。

## 1. 管理后台 `admin-ui/`

- 技术栈：Vue 3、Vite、Pinia、Element Plus、Axios
- 本地启动：

```bash
cd admin-ui
npm install
npm run dev
```

- 默认通过 `/admin` 提供页面，通过 `/admin/api` 访问后台接口
- 本地开发由 `vite.config.js` 将 `/admin/api` 代理到后端
- 生产环境由 `backend/app.js` 托管 `admin-ui/dist`
- 路由权限以 `meta.permission` 为入口，必须与 `backend/config/adminPermissionCatalog.js` 保持一致

## 2. 微信小程序 `miniprogram/`

- 技术栈：微信原生小程序、WXML、WXSS、Page
- 本地启动：使用微信开发者工具打开 `miniprogram/`
- 主要页面入口见 `miniprogram/app.json`
- 当前主业务覆盖：首页、分类、活动、购物车、用户中心、订单、自提、分销中心、钱包、积分、抽奖、优惠券、砍价、拼团
- 接口基址和环境切换以 `miniprogram/config/env.js`、`miniprogram/config/constants.js` 为准

## 3. 关于代理门户

当前仓库里没有 `agent-portal/` 前端目录，不应再把它视为当前前端子项目。

当前真实状态是：

- 后端仍保留 `/api/portal/*` 门户接口
- 小程序仍保留“门户初始密码申请”等链路
- 如果后续恢复门户前端，应单独确认源码位置和部署方式，不要根据旧文档假设其仍在本仓库
