# 业务规则（术语 · 佣金 · 商品与 SKU）

---

## 一、术语表

### 价格相关

| 名称 | 含义 |
|------|------|
| 零售价 | 商品对外基础标价，通常对应 `Product.retail_price`。 |
| 市场原价 | 划线价，可选，用于展示优惠力度。 |
| 成本价 | 平台进货/结算基准，下单与佣金池常与此关联。 |
| 会员价 / 团长价 / 代理价 | 不同身份看到的价格档位（`price_member` 等）。 |
| SKU | 同一商品下的规格行（颜色、尺码等），各有库存与可选独立售价。 |

### 佣金相关

| 名称 | 含义 |
|------|------|
| 商品级独立佣金 | 开启「独立佣金设置」后，仅对该商品生效的比例/固定金额。 |
| 直推 / 间推 | 一级上级 / 二级上级获得的佣金。 |
| **比例字段小数约定** | 后端 `CommissionService` 使用 **`实际金额 × rate`，`rate` 为 0～1 小数**（如 `0.2` = 20%）。管理后台商品表单填 **0～100 百分数**，保存时换算为小数入库。 |

### 系统设置里的「分销佣金比例」

与「商品级直推/间推比例」**不是**同一套存储键：设置页写入 `AppConfig` 等；具体参与哪些计算以 `CommissionService._getConfig()` 与保存逻辑为准。

## 二、佣金规则（研发 / 运营）

订单佣金分配在 `backend/services/CommissionService.js`。

- **比例佣金**：`金额 = 实际支付价 × commission_rate`，**`commission_rate` 在库中为 0～1 小数**。
- **管理后台商品页**：展示百分数 0～100；提交前前端除以 100；后端 `normalizeProductCommissionRate` 对大于 1 的入参按百分数再除以 100 兜底。
- **固定金额**：`commission_amount_1` / `commission_amount_2` 等为**元**，且**优先于比例**（见服务内分支）。
- **系统设置 Tab「分销佣金比例」**：写入 `AppConfig`，与商品表 `commission_rate_1` 等不是同一字段；改全局策略时请对照 `CommissionService` 与设置保存逻辑。

**相关代码**：`CommissionService.js`、`commissionRates.js`、`adminProductController.js`。

## 三、商品、SKU 与价格

### 商品（Product）

维护名称、主图/详情图、类目、**商品层价格档位**（零售、会员、团长、代理等）与**总库存**等。未启用多规格时，可主要用商品层价格与库存。

### SKU（`product_skus`）

一行一个可售卖规格。可配置**规格价、规格库存**；下单、购物袋通常带 `sku_id`。启用多规格后，后台需维护完整 SKU；更新商品时接口会**整批替换** SKU。

### 与展示价的关系

用户端展示价结合商品配置、SKU、用户角色、购买层级等，由 `PricingService` 等计算，不单以某一字段为准。
