# 管理后台快速启动指南

## 已完成的工作

✅ 管理后台已经创建完成并构建成功！

项目位置：`backend/admin-ui/`

技术栈：
- Vue 3 + Composition API
- Element Plus UI 组件库
- Vite 5 构建工具
- Pinia 状态管理
- Vue Router 4 路由

## 如何访问管理后台

### 方式一：直接访问（推荐）

1. 确保后端服务正在运行：
   ```bash
   cd backend
   npm run dev
   # 或
   npm start
   ```

2. 浏览器访问：
   ```
   http://localhost:3000/admin/
   ```

### 方式二：开发模式（用于开发调试）

1. 启动后端服务（端口 3000）：
   ```bash
   cd backend
   npm run dev
   ```

2. 在新终端启动前端开发服务器：
   ```bash
   cd backend/admin-ui
   npm run dev
   ```

3. 浏览器访问：
   ```
   http://localhost:5173/admin/
   ```

## 登录管理后台

### 1. 创建管理员账号

首次使用需要创建管理员账号：

```bash
cd backend
node scripts/create-admin.js
```

按照提示输入：
- 用户名（例如：admin）
- 密码（至少6位）
- 姓名（例如：管理员）

### 2. 登录

使用刚创建的账号登录管理后台。

## 功能模块

登录后可以使用以下功能：

1. **数据概览** - 查看系统统计数据
2. **商品管理** - 管理商品（增删改查、上下架）
3. **订单管理** - 查看订单、订单发货
4. **用户管理** - 管理用户角色
5. **提现审核** - 审核用户提现申请
6. **售后管理** - 处理售后申请
7. **系统设置** - 查看系统信息、修改密码

## 重新构建管理后台

如果修改了前端代码，需要重新构建：

```bash
cd backend/admin-ui
npm run build
```

构建完成后，重启后端服务即可看到更新。

## 常见问题

### 1. 访问 /admin/ 显示 404

**解决方案**：
- 确保已运行 `npm run build` 构建前端
- 确保后端服务已重启
- 检查 `backend/admin-ui/dist/` 目录是否存在

### 2. 登录后 API 请求失败

**解决方案**：
- 检查后端服务是否正常运行
- 查看浏览器控制台错误信息
- 确认数据库连接正常

### 3. 无法登录

**解决方案**：
- 确认管理员账号已创建
- 检查用户名密码是否正确
- 查看后端日志确认错误信息
