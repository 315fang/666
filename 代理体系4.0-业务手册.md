# 代理体系业务说明（当前入口）

本文件名保留用于兼容历史引用，但原“4.0 手册”中的阶段性说法已经不再代表当前真实实现。

当前代理体系的有效业务口径请优先看：

- [`docs/architecture/项目业务总览.md`](./docs/architecture/项目业务总览.md)
- [`docs/业务规则.md`](./docs/业务规则.md)
- `backend/config/constants.js`
- `admin-ui/src/views/agent-system/index.vue`
- `admin-ui/src/views/n-system/index.vue`

## 1. 当前真实角色体系

### 1.1 标准路径

当前代码中的标准角色仍是：

- `0`: 普通用户
- `1`: C1 初级代理
- `2`: C2 高级代理
- `3`: B1 推广合伙人
- `4`: B2 运营合伙人
- `5`: B3 区域合伙人

### 1.2 N 路径

当前代码中的 N 路径是独立角色，不是拿 B2/B3 直接改名：

- `6`: 小 n 代理
- `7`: 大 N 独立代理

## 2. 当前业务要点

### 2.1 标准路径并非“只靠固定价差”

当前佣金实现同时包含：

- 后台可配置的百分比佣金
- 商品级佣金覆盖
- 固定金额类协助奖
- 代理发货利润

不能再把当前实现理解成“只靠价差、无需比例配置”。

### 2.2 N 路径是单独一条经营链

当前已经落地的 N 路径能力包括：

- 邀约入场
- 入场申请与审核
- 小 n 向大 N 申请货款
- 大 N 主动划拨货款
- 小 n 升大 N
- 价差归集到大 N 可提现余额
- 升级脱离奖励

### 2.3 业务配置主要落在后台和配置表

当前代理体系相关配置主要由以下位置共同决定：

- 后台“代理体系”页面
- 后台“N 路径代理”页面
- `app_configs` 中 `agent_system_*` 配置
- 后端常量与服务层默认值

## 3. 当前不该继续沿用的旧理解

- 不再把 N 路径理解为“复用 B2/B3 的改名版”
- 不再把所有佣金规则理解为固定金额
- 不再把旧手册中的阶段性金额示例当成当前唯一真相

## 4. 如果要改代理业务

1. 先确认改动属于标准路径还是 N 路径
2. 同时核对 `backend/config/constants.js`、服务层和后台配置页
3. 同步更新 `docs/architecture/项目业务总览.md` 与 `docs/业务规则.md`
