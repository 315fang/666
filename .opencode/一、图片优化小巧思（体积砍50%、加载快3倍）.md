### **一、图片优化小巧思（体积砍50%、加载快3倍）**
1. **CDN + WebP + 自动降质命名规则**（京东电商标配）  
   后端图片服务支持`?imageMogr2/format/webp/quality/70` 或命名`!q70`。前端`<image src="{{item.pic}}?imageMogr2/format/webp"/>`。  
   京东实测：单张从800KB→180KB，首屏图片加载从1.8s→0.6s。牛在：零代码改动，后端一键控制所有图片质量。

2. **image组件 lazy-load + IntersectionObserver 预加载关键图**（腾讯课堂）  
   ```wxml
   <image lazy-load="true" src="{{url}}"/>
   ```
   + 用IntersectionObserver提前监听“即将进入可视区”的图片，提前wx.downloadFile缓存。  
   腾讯课堂长图文页：滑动卡顿消失，图片加载成功率99.9%。

3. **雪碧图 + Base64小图标 + WebP大图混合**（阿里本地生活）  
   小图标全转Base64塞到CSS变量，后端返回雪碧图URL。大厂实测：包体积再砍8-12%。

### **二、页面 + 分包优化（启动提速70%）**
4. **独立分包 + 主包极简**（京东京喜首页实战）  
   app.json里加`"independent": true`，把高频启动页（首页、活动页）做独立分包。  
   京东：独立分包启动不等主包，首屏从2.3s→0.8s。牛在：广告/支付页用户根本感觉不到“下载”。

5. **分包预下载 + preloadRule**（腾讯官方+阿里）  
   app.json：
   ```json
   "preloadRule": {
     "pages/index/index": { "packages": ["__APP__", "packageA"] }
   }
   ```
   用户进首页时，微信自动在后台下载后续分包。腾讯课堂二次进入秒开。

6. **Taro.preload / wx.preload 跳转前置数据**（京东购物车）  
   ```js
   // A页
   Taro.preload({ goodsList: data })
   Taro.navigateTo({ url: '/pages/B' })
   // B页 onLoad
   const preload = Taro.getCurrentInstance().preloadData
   ```
   京东商详页：跳转白屏时间从300ms→几乎0，实测提速28%。

### **三、不同计算逻辑优化（卡顿归零）**
7. **WXS视图层计算**（腾讯内部40%渲染提升）  
   把过滤、格式化、简单数学丢到.wxs文件，视图层直接跑，不走逻辑层setData。  
   ```wxs
   var formatPrice = function(price) { return (price/100).toFixed(2) }
   ```
   腾讯视频号列表：setData次数砍70%，FPS稳60。

8. **Worker线程卸载重计算**（字节电商）  
   ```js
   const worker = wx.createWorker('workers/calc.js')
   worker.postMessage({ data: heavyData })
   ```
   图片压缩、复杂排序全丢Worker，主线程不卡。字节10万条数据列表丝滑。

### **四、加载优化（用户永远感觉“在加载但不卡”）**
9. **骨架屏 + 占位组件动态切换**（腾讯课堂全流程）  
   onLoad先渲染`<skeleton />`，数据回来用`<block wx:if="{{loaded}}">真实内容</block>`无缝替换。  
   用户感知加载时间缩短60%，留存+15%。

10. **初始渲染缓存 + 数据预拉取**（微信官方+京东）  
    app.json加`"initialRenderingCache": "static"`，微信帮你缓存首屏HTML。配合App.onLaunch就拉首页数据。

### **五、缓存小巧思（请求减半）**
11. **本地缓存智能双层**（京东结算页）  
    ```js
    const cache = wx.getStorageSync('key')
    if (cache && Date.now() - cache.time < 5*60*1000) return cache.data
    // 否则请求 + wx.setStorageSync({data, time: Date.now()})
    ```
    京东：搜索/筛选5分钟内零请求。

12. **请求防抖 + 接口聚合**（阿里系）  
    多接口并成一个，后端返回大JSON，前端递归拆解只setData变化部分。请求数↓70%。

### **六、跳转 + 搜索优化**
13. **搜索防抖 + 搜索历史本地缓存**（字节搜索页）  
    ```js
    onInput: debounce(function(e){ this.search(e.detail.value) }, 300)
    ```
    历史用wx.getStorageSync，搜索即显示，字节实测输入体验“像原生App”。

### **七、用户感受极致优化**
14. **静默更新 + 分级强更**（京东大促）

### **八、后端运行 + 修正前端 + 控制页面（大厂最牛逼的部分！）**
15. **云开发Watch实时推送**（腾讯云原生）  
    ```js
    db.collection('data').where(...).watch({ onChange: res => this.setData({list: res.docs}) })
    ```
    后端改数据，前端实时更新，无轮询！腾讯文档/视频号天天用。

16. **后端返回CSS变量动态主题**（阿里/腾讯组件库）  
    后端接口返回：
    ```json
    { "theme": { "--primary-color": "#ff0000", "--bg": "#f5f5f5" } }
    ```
    前端：
    ```js
    wx.setStorageSync('theme', data.theme)
    document.documentElement.style.setProperty('--primary-color', data.theme['--primary-color'])
    ```
    支持用户/运营后台一键换肤，阿里本地生活已上线3套主题。

17. **后端配置页面样式模板**（京东A/B测试）

    京东：灰度10%用户先上线新样式，翻车秒回滚。

    阿里：运营后台一键切换“双11皮肤” vs “日常皮肤”。
