- 仅允许修改 qianduan/ 目录内文件。
- 禁止修改 backend/、backend/admin-ui/、docs/ 及仓库根文档，除非用户明确要求。

2. 防重复代码
- 新增任何函数、组件、页面逻辑前，先检索现有实现：
  rg "<关键词>" qianduan
- 优先复用已有能力：qianduan/utils、qianduan/components、qianduan/store。
- 禁止复制粘贴已有业务逻辑到新文件。

3. 防重复文档
- 默认不新增 markdown 文档。
- 禁止生成 SUMMARY/REPORT/PLAN 一类重复文档。
- 如需文档说明，优先在现有文件中增量更新。

4. 变更原则
- 只做与任务直接相关的最小改动。
- 保持现有代码风格（CommonJS、小程序原生写法）。
- 不得臆造 API 或配置；不确定时先读取现有代码。

5. 输出要求
