# 商品自由选套餐需求

## 背景

当前仓库已经存在一套 `product_bundles` 组合商品能力：

- 后台可配置组合标题、组合价、分组、候选商品、默认规格、每组最少/最多选择数
- 小程序已有组合详情页与组合下单链路
- 订单侧已支持组合价、组合优惠分摊、组合订单禁用优惠券/积分、整套退款

但当前产品形态更偏“产品组合”，还没有被包装成用户更易理解的“自由选套餐”：

- 套餐入口命名不够直观
- 后台配置语义偏技术化，不能清晰表达“套餐一 / 套餐二 / 自由搭配”
- 前台没有把“任选几个、再选几个、最终固定套餐价”作为核心卖点展示
- 缺少适合运营投放的套餐模板视角

## 目标

在现有 `product_bundles` 能力基础上，收口出一套“商品自由选套餐”形态，使运营可以像配置麦当劳套餐一样配置：

1. 创建一个固定套餐价，例如 `399`
2. 将套餐拆成多个选择分组，例如：
   - 洁面：任选 `1` 件
   - 精华：任选 `2` 件
   - 面膜：任选 `3` 件
3. 用户在小程序前台按分组完成选择
4. 系统始终按套餐固定价结算，而不是所选商品逐件原价累加支付

## 范围

- `cloudfunctions/admin-api/src/admin-product-bundles.js`
- `admin-ui/src/views/product-bundles/index.vue`
- `cloudfunctions/order/product-bundle.js`
- `cloudfunctions/order/order-create.js`
- `miniprogram/pages/product-bundle/*`
- 必要的分类页/活动页/入口文案与命名调整
- 必要的文档与数据结构收口

## 非目标

- 不新增“组合订单可部分退款”
- 不允许组合订单叠加优惠券、积分、拼团、砍价、限时活动
- 不实现复杂动态计价（例如多选一件加 0 元，多选第二件再加 20 元）
- 不引入多商家混合履约

## 用户故事

### Story 1

作为运营，我希望在后台创建一个“自由选套餐”，设置固定套餐价和每个分组的可选商品与选择数量，这样可以快速组合出套餐一、套餐二、爆款套餐等销售模板。

### Story 2

作为用户，我希望在一个套餐页里直观看到“这组选几个、那组选几个”，完成选择后直接按套餐价购买，这样比逐个单品自己凑单更清晰。

### Story 3

作为客服/售后，我希望自由选套餐下单、支付、订单详情、退款口径都与当前组合商品保持一致，这样不会额外引入新的售后复杂度。

## 业务规则

1. 自由选套餐本质上仍属于组合订单，沿用当前 `bundle` 订单类型。
2. 套餐必须有固定套餐价，且套餐价不得高于所选商品原价合计。
3. 每个分组都必须配置：
   - 分组名称
   - 最少选择数
   - 最多选择数
   - 候选商品列表
4. 每个候选商品可绑定默认规格；未绑定时沿用商品默认规格。
5. 用户只有在所有分组都满足最少/最多选择规则后，才允许下单。
6. 套餐订单禁止叠加：
   - 优惠券
   - 积分抵扣
   - 拼团
   - 砍价
   - 限时活动
   - 兑换订单
7. 套餐订单售后仍按“整套退款”处理。

## 待收口的产品决策

### 决策 1：套餐模板层级

需要明确“套餐一 / 套餐二”是：

- 方案 A：每个套餐就是一条独立 `product_bundle`
- 方案 B：一个“套餐活动”下挂多个 `product_bundle`

当前默认按 **方案 A** 处理，即每个套餐一条独立组合记录，先不引入新的“套餐活动”父层级。

### 决策 2：分组数量与展示

当前默认支持任意数量分组，但前台优先按 3-5 组的套餐视觉来设计，避免单页过长。

### 决策 3：套餐入口命名

当前默认将“产品组合”对用户侧逐步收口为“自由选套餐”或“套餐专区”，后台保留兼容字段与接口，不强行重命名数据库集合。

## 验收标准

### AC1

When 运营在后台创建一条自由选套餐, the system shall 支持设置固定套餐价、套餐标题、副标题、封面、多个选择分组与候选商品。

### AC2

When 运营为某个分组设置最少选择数与最多选择数, the system shall 拒绝保存不合法配置，例如最多选择数大于候选商品数。

### AC3

When 用户进入自由选套餐详情页, the system shall 展示套餐固定价、已选数量、套餐原价合计与套餐优惠金额。

### AC4

When 用户尚未完成全部必选分组, the system shall 禁止下单并明确提示“请选择完整组合”。

### AC5

When 用户完成全部选择并提交订单, the system shall 生成 `bundle` 订单，并按套餐固定价结算。

### AC6

When 自由选套餐订单进入结算页, the system shall 禁止使用优惠券、积分与其他活动优惠，并展示对应禁用提示。

### AC7

When 自由选套餐订单发起售后, the system shall 仅支持整套退款，不允许用户单独退其中某个子商品。

### AC8

When 运营需要配置“套餐一 / 套餐二 / 套餐三”, the system shall 允许通过多条独立套餐记录实现，而不要求新增父级套餐活动模型。
