## 📌 核心业务流程

### 1. 用户系统

#### 1.1 用户生命周期

```
新用户
  ↓
[微信登录] → openid 创建
  ↓
[完善信息] → 姓名、头像、电话
  ↓
[正常使用] → 可以购物、参与分销
  ↓
[积分成长] → 等级提升、权益增加
  ↓
[账号注销] → 数据归档、账号禁用
```

#### 1.2 用户等级系统

**等级划分** (成长值驱动)

| 等级 | 名称 | 成长值 | 权益 | 备注 |
|------|------|--------|------|------|
| 1 | 新手 | 0-499 | 新人券、基础折扣 | 默认 |
| 2 | 青铜 | 500-1999 | 额外积分、生日礼 | 消费达标升级 |
| 3 | 白银 | 2000-4999 | VIP 折扣、免运费 | 累计消费或邀请 |
| 4 | 黄金 | 5000-9999 | 专享商品、返利 | 高价值用户 |
| 5 | 铂金 | 10000+ | 礼遇、顾问服务 | VIP 待遇 |

**成长值获取途径**
- 购物: 消费 1 元 = 1 成长值
- 邀请: 邀请成功 = 100 成长值
- 评价: 商品评价 = 10 成长值
- 活动: 参与营销活动 = 50-200 成长值

**权益说明**
- 积分倍数: 高等级用户获得积分倍数增加
- 优惠券: 高等级用户获得更好的优惠券
- 免运费: 白银及以上享受免运费
- 生日礼: 在生日月份获得专项礼物

#### 1.3 积分与钱包

**积分系统** (虚拟货币)
- 获得途径: 购物、评价、活动、邀请
- 使用方式: 可用于兑换优惠或抵扣
- 有效期: 2 年无活动自动清零
- 兑换比率: 100 积分 = 1 元

**钱包余额** (真实货币)
- 充值: 用户可手工充值
- 收入: 退货退款、分销佣金
- 支出: 购物支付、提现
- 最低提现: 100 元
- 提现手续费: 2%

### 3. 购物车与订单

#### 3.1 购物流程

```
浏览商品
  ↓ (选择规格)
[添加到购物车]
  ├─ 购物车持久化 (云数据库)
  ├─ 库存预占 (可选)
  └─ 推送提示
  
选择商品
  ↓ (勾选)
[进入结算]
  ├─ 验证库存
  ├─ 验证优惠券
  ├─ 计算价格
  └─ 计算运费
  
确认信息
  ↓ (地址、优惠)
[生成订单]
  ├─ 订单号生成
  ├─ 库存冻结
  ├─ 积分计算
  └─ 分销佣金计算
  
支付订单
  ↓ (微信支付)
[订单确认]
  ├─ 库存扣减
  ├─ 金额入账
  ├─ 通知商家
  └─ 通知用户
  
发货与收货
  ↓
[订单完成]
  ├─ 物流更新
  ├─ 确认收货
  └─ 可以评价
```

#### 3.2 订单状态管理

```
订单生命周期

创建订单
    ↓
待支付 (pending_payment)
    ├─ 支付成功 → 已支付 (paid)
    │              ↓
    │         待发货 (pending_ship)
    │              ↓
    │         已发货 (shipped)
    │              ↓
    │         待收货 (in_transit)
    │              ↓
    │         已完成 (completed)
    │              └─ 可以评价 (7 天内)
    │
    └─ 支付失败/超时 → 已取消 (cancelled)

特殊情况:

已完成 → 申请售后 → 退货中 (in_refund)
                    └─ 已退货 (refunded)

已支付 → 订单被动取消 (by_system)
```

**各状态规则**:
- **待支付**: 15 分钟后自动取消
- **待发货**: 商家需在 48 小时内发货
- **已发货**: 自动确认收货时间 (7-15 天)
- **已完成**: 可在 7 天内申请退货

#### 3.3 价格计算逻辑

```
订单总价 = 商品小计 + 运费 - 优惠 + 税费

商品小计计算:
  for each item in cartItems {
      if (hasSkuPrice) {
          unitPrice = sku.price
      } else {
          unitPrice = product.retailPrice
      }
      
      if (userLevel >= 3) {  // 白银及以上
          unitPrice = unitPrice * 0.95  // 95 折
      }
      
      itemTotal += unitPrice * quantity
  }

优惠计算:
  if (coupon) {
      discount = coupon.value
      if (itemTotal < coupon.minPurchase) {
          throw error("不满足优惠券使用条件")
      }
  }

运费计算:
  if (userLevel >= 3) {  // 白银及以上免运费
      shippingFee = 0
  } else if (itemTotal >= 100) {  // 满 100 包邮
      shippingFee = 0
  } else {
      shippingFee = 10  // 默认 10 元
  }

税费:
  taxFee = 0  // 国内一般不计税

总价:
  total = max(1, itemTotal - discount + shippingFee + taxFee)
```

#### 3.4 库存管理

```
库存状态:
  总库存 (total_stock)
    = 可售库存 + 预占库存 + 冻结库存 + 缺货库存

库存操作流程:

1. 用户加入购物车
   → 库存不变 (仅在本地记录)

2. 生成订单
   → 可售库存 -= qty
   → 冻结库存 += qty

3. 订单支付成功
   → 冻结库存 -= qty
   → 出货库存 += qty (可选)

4. 订单发货
   → 出货库存 -= qty

5. 订单取消 / 退款
   → 可售库存 += qty
   → 冻结库存 -= qty

库存预警:
  if (availableStock < 50) {
      warn("库存即将不足")
  }
  if (availableStock == 0) {
      setStatus("out_of_stock")
      notify("商品已下架")
  }
```

### 6. 分销系统

#### 6.1 分销结构

```
分销模式: 两级分销 (即 S2B2C)

供应商 (Supplier)
  │
  ├─ 一级分销商 (Distributor Level 1)
  │  │
  │  ├─ 下属消费者 (Consumer)
  │  │  └─ 产生订单 → 一级获得佣金
  │  │
  │  └─ 下属二级分销商 (Distributor Level 2)
  │     └─ 下属消费者 → 一级、二级各获得佣金
  │
  └─ 消费者 (Consumer)
     └─ 产生订单 → 供应商收入
```

#### 6.2 分销佣金计算

```
商品分销配置:
  commission_rate = 15%  // 以商品零售价为基准

订单佣金分配:

订单包含商品 A (零售价 100 元，佣金 15%)

情景 1: 消费者直接购买
  └─ 供应商收入: 100 元
  
情景 2: 一级分销商的邀请用户购买
  ├─ 供应商收入: 100 元 - 15 元佣金
  └─ 一级分销商获得: 15 元 (15% × 100)

情景 3: 二级分销商的邀请用户购买
  ├─ 供应商收入: 100 元 - 15 元佣金
  ├─ 一级分销商获得: 12 元 (80% × 15)
  └─ 二级分销商获得: 3 元 (20% × 15)

佣金提现:
  ├─ 累计佣金 ≥ 100 元
  ├─ 实际到账 = 佣金 × (1 - 2% 手续费)
  └─ T+1 结算 (工作日)
```

#### 6.3 分销资格

```
升级为分销商:
  ├─ 用户等级 ≥ 白银 (2000 成长值)
  ├─ 累计消费 ≥ 500 元
  └─ 通过实名认证

维持分销资格:
  ├─ 6 个月内有邀请
  ├─ 6 个月内有佣金收入
  └─ 账号处于正常状态

等级提升:
  分销商 → 优秀分销商 (年佣金 > 10000 元)
          → 超级分销商 (年佣金 > 50000 元)
          → 合伙人 (年佣金 > 100000 元)

权益对应:
  ├─ 优秀分销商: 额外 2% 佣金奖励
  ├─ 超级分销商: 额外 5% 佣金奖励
  └─ 合伙人: 额外 8% 佣金奖励 + 专属客服
```

### 9. 数据安全与合规

#### 9.1 数据保护

```
用户数据安全:
├─ 密码加密: bcrypt + salt
├─ 敏感信息: 电话号码、身份证加密存储
├─ 日志记录: 所有操作记录用户、时间、行为
├─ 访问控制: 仅管理员可查看用户隐私信息
└─ 数据备份: 每日备份，保留 30 天

支付数据安全:
├─ PCI DSS 合规
├─ 不存储银行卡信息
├─ HTTPS 加密传输
├─ 签名验证: 所有支付通知都需验证签名
└─ 审计日志: 所有支付记录可审计
```

#### 9.2 反作弊

```
异常检测:
├─ 频繁操作: 5 分钟内操作超过 50 次 → 限流
├─ 异常登录: IP 地址突变 → 验证
├─ 恶意刷单: 同一用户 1 小时内下单超过 10 笔 → 冻结
├─ 优惠券滥用: 批量领取 → 暂停账户
└─ 分销欺诈: 自我邀请 → 取消分销资格

风控规则:
├─ 新账户 24 小时内消费限额 1000 元
├─ 未认证账户分销功能禁用
├─ 异常操作后需手机验证
└─ 频繁退款账户需人工审核
```
