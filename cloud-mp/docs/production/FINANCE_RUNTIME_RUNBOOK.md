# 资金运行手册

## 退款
- 内部退款误退后，优先使用 `npm run repair:refund-reopen` 对问题单做 dry-run。
- 若货款余额不足，允许使用“转欠款”模式回退，不允许硬扣成负数。
- 回退完成后，退款单必须回到 `approved`，并补齐反向流水。

## 欠款
- 财务在“财务看板 -> 代理商货款欠款”中处理欠款。
- 处理方式仅允许：
  - 货款抵扣
  - 线下确认
- 每次处理都必须填写原因。

## 提现
- 审核通过不代表已打款。
- 拒绝提现必须填写拒绝原因。
- 确认打款必须填写打款备注。

## 货款支付
- 货款支付是内部资金动作，任何订单状态或流水写失败都必须回滚。

## 佣金
- 佣金结算、佣金转货款都必须留资金流水。
- 若流水失败，必须回滚佣金余额变更。
