# 后台值守手册

日期：2026-04-17

## 1. 使用原则

值守目标不是直接修代码，而是先判断：

1. 问题影响什么链路
2. 后台是否还能继续处理业务
3. 是否需要升级给研发 / 财务 / 发布负责人
4. 是否达到回滚判断条件

## 2. 后台登录异常

### 先看什么

- `运维监控 -> 近期错误日志`
- `system/status`
- 当前环境与发布记录

### 去哪里处理

- 后台登录页
- 运维监控页

### 成功判据

- 管理员账号可登录
- 登录后菜单与权限正常

### 失败后怎么升级

- 连续登录失败或 401/500 异常上升，立即升级给研发负责人
- 若影响后台登录整体可用性，进入回滚评估

## 3. 订单状态异常

### 先看什么

- 订单详情页
- 订单链路审查
- 最近支付/退款/发货动作

### 去哪里处理

- `订单管理`
- `运维监控 -> 订单链路审查`

### 成功判据

- 订单状态与实际业务动作一致
- 后台列表与详情一致

### 失败后怎么升级

- 若状态长时间卡住或批量异常，升级给研发与值班负责人

## 4. 退款处理中或回写异常

### 先看什么

- 退款单状态
- 微信退款状态同步结果
- 订单链路与用户资金链

### 去哪里处理

- `售后退款`
- `运维监控 -> 订单链路审查`
- `运维监控 -> 用户资金链路`

### 成功判据

- 退款状态进入终态
- 订单、佣金、库存、余额链路一致

### 失败后怎么升级

- 退款长时间停留 `processing`
- 退款金额与订单金额不一致
- 出现重复退款或资金链不一致

以上任一情况，立即升级给资金域负责人和研发负责人。

## 5. 提现打款确认

### 先看什么

- 提现单状态
- 打款备注
- 实际打款渠道结果

### 去哪里处理

- `提现审核`

### 成功判据

- 审核通过后，打款完成才允许改终态
- 提现备注已记录

### 失败后怎么升级

- 打款失败或备注缺失，交由财务负责人确认，不允许值班人员自行猜测完结

## 6. 配置来源核查

### 先看什么

- 当前页面展示值
- 配置来源审查结果

### 去哪里处理

- `运维监控 -> 配置来源审查`
- 对应配置页面

### 成功判据

- 生效值、配置来源、后台展示一致

### 失败后怎么升级

- 若 singleton / configs / app_configs 冲突，升级给配置负责人

## 7. 素材 / 上传 / 存储异常

### 先看什么

- 对象存储状态
- 素材上传错误信息
- 最近发布与配置修改记录

### 去哪里处理

- `素材管理`
- `运维监控 -> 对象存储`

### 成功判据

- 上传成功
- 预览链接可展示
- 素材在后台和小程序能正常读取

### 失败后怎么升级

- 持续上传失败或存储探测失败，升级给研发和值班负责人

## 8. 欠款与余额处理

### 先看什么

- 用户当前货款余额 / 欠款
- 最近流水
- 处理原因

### 去哪里处理

- `财务看板`
- `用户管理`
- `运维监控 -> 用户资金链路`

### 成功判据

- 处理方式明确
- 原因明确
- 流水与余额一致

### 失败后怎么升级

- 若余额、欠款、提现、佣金间出现链路不一致，交由资金域负责人和研发处理

## 9. 值班责任

- 发布执行人：负责发布动作和发布记录
- 值班负责人：负责观察窗口和异常分流
- 资金域负责人：负责退款、提现、欠款、余额问题裁决
- 回滚决策人：负责是否进入回滚

责任人未明确时，不允许进入发布窗口。
