# 修复文档模板

文件命名：

`YYYY-MM-DD-<topic>.md`

---

# 标题

日期：YYYY-MM-DD

## 1. 问题描述

说明用户看到的问题、链路断点或字段失真现象。

## 2. 影响范围

列出受影响的模块、页面、云函数、后台接口或数据集合。

## 3. 根因

说明问题为什么会发生。  
优先写字段真相差异、兼容逻辑扩散、writer / reader 不一致、页面本地猜字段等真实根因。

## 4. 修复方案

按顺序说明：

1. contract / normalizer / assembler 如何调整
2. writer 侧如何修
3. reader / page / admin consumer 如何收口
4. 哪些 fallback 被删除或降级

## 5. 兼容策略

明确：

- 哪些旧字段暂时保留
- 哪些旧字段只读兼容
- 哪些旧字段禁止新增
- 是否涉及 repair 脚本或人工修复步骤

## 6. 改动模块

只列关键模块，不要写成逐文件流水账。

## 7. 验证命令与结果

列出实际执行的命令和结果。  
如果没有执行某项验证，必须写明原因。

## 8. 回归风险

说明本轮修复可能影响的链路和剩余观察点。

## 9. 后续待清债务

列出这轮没有处理、但必须继续追的技术债。
