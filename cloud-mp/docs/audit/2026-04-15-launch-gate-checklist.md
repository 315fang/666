# 上线前业务逻辑闸口清单

日期：2026-04-15

## 1. 区域奖励归属口径

- 区域奖励只按订单收货地址中的 `province / city / district` 精确命中。
- 命中优先级固定为：`district > city > province`。
- 历史 `school` / `area` 类型不再参与区域奖励结算。
- 区域累计金额按同一主体下全部命中订单的实付金额汇总后套阶梯：
  - `10万 = 1%`
  - `30万 = 2%`
  - `100万 = 3%`

## 2. 虚拟 B3 结算主体

上线前必须在 `users` 中准备一条内部结算用户，并填写以下字段：

```json
{
  "is_virtual_settlement": true,
  "virtual_settlement_type": "b3_region",
  "virtual_display_name": "虚拟B3结算主体"
}
```

说明：

- 该用户只作为佣金接收主体使用，不参与小程序登录。
- 绑定方式为后台“区域归属”页填写 `claimant_id`。
- 运行时落账类型为 `region_b3_virtual`，落账等级固定按 `5`。

## 3. 兑换券订单口径

- `coupon_type='exchange'` 表示兑换资格券，不是普通满减券。
- 兑换券订单 `type='exchange'`，规则固定：
  - 实付 `0 元`
  - 不参与普通优惠券
  - 不可积分抵扣
  - 不产生成长值
  - 不产生复购积分
  - 不触发分销佣金、区域奖励、自提奖励、协助奖、代理发货利润
  - 允许正常发货、收货、售后
- 一张兑换券默认只允许兑换一件商品。
- 历史兑换券如无 `exchange_meta`，需要先执行后台“补绑历史兑换券”再使用。
