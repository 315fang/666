# 测试账号、清数与分权运维说明

## 蔚总或验收用「顶级会员 + 高佣金」测试

1. 使用**管理后台**「用户管理」将目标账号 `role_level` 设为 **5（区域合伙人）** 或 **6（线下实体门店）**，视业务需要而定；勿在生产环境长期保留测试用超高权限。
2. **佣金比例**：在「财务看板 / 代理体系」或 `agent-system` 相关配置中调整规则；具体入口以当前后台菜单为准。
3. **测试结束后**：在 CloudBase 控制台或通过后台删除/匿名化**测试订单、测试佣金流水**；若已有真实用户下单混入数据，只能**部分清理**，可配合后台筛选隐藏测试单（需在订单上打标或单独测试环境）。

## 分权账号与操作留痕

1. 在 **管理员与权限**（`/admins`）为每个岗位创建**独立子账号**，勿多人共用超级管理员。
2. 敏感操作依赖 **操作日志**（`/logs`）留痕；要求每人使用自己的账号登录后台。

### 推荐后台子账号基线

建议至少拆为以下 6 个岗位账号：

| 用户名 | 角色 | 典型职责 |
| --- | --- | --- |
| `ops01` | `operator` | 商品、活动、页面、群发、日常运营 |
| `finance01` | `finance` | 提现、佣金、财务核对 |
| `service01` | `customer_service` | 订单、售后、用户查询 |
| `warehouse01` | `warehouse` | 发货、自提、库存协同 |
| `design01` | `designer` | 页面素材、图文内容 |
| `channel01` | `channel_manager` | 经销商、分支代理、B1 定向邀约 |

说明：

1. `admin` / `super_admin` 只保留给极少数负责人，不再作为日常共用账号。
2. 新账号首次登录后应立即在后台修改密码。

## 小程序侧优惠券数量一致

「我的」页与首页未使用券数量均来自 `/coupons/mine?status=unused`，并在前端**过滤已过期**券，避免与活动页展示不一致。若仍不一致，请核对云函数返回的 `expire_at` 字段是否准确。

## 游客账号、测试数据与后台清理

### 当前原则

1. **不要直接批量删生产数据**。先隐藏、再观察、最后只删除无业务关系的游客账号。
2. **游客账号**按以下条件识别：`role_level = 0`，昵称为空 / `新用户` / `微信用户`，无头像、无手机号，且注册超过 24 小时。
3. **有业务关系的不清理**：已有订单、佣金、提现、货款流水、定向邀约、上下级关系的用户必须保留。
4. **有外围数据的不删除**：已隐藏用户若还有地址、购物车、优惠券、积分、消息等记录，先保留，避免留下孤儿数据。
5. **测试订单优先打标，不直接删除**。后台订单页支持“标记为测试订单”，标记后默认从常规列表和统计口径中排除。

### 推荐执行顺序

1. 在后台「用户管理」默认不勾选“含隐藏账号”，日常运营只看有效用户。
2. 对明显测试订单，在「订单管理」里先标记为测试订单；不要混在真实业务统计里。
3. 对无授权游客账号，使用 `visitor-account-cleanup` 云函数处理：
   - 预演：调用函数并传 `{ "dryRun": true }`，只看会隐藏/删除多少，不改数据。
   - 首次执行：调用函数并传 `{ "mode": "hide_only" }`，只隐藏游客账号。
   - 稳定后：配置每日定时触发，默认只执行隐藏模式，不自动删除。
4. 对 `_bak` 备份集合，先确认有可恢复备份，再按集合分批清理；不要和游客账号清理混为一件事。
