# 管理后台快速启动

更新日期：2026-04-18

本文件只说明当前 `cloud-mp/admin-ui` 的启动方式，不再使用旧 `backend/admin-ui` 路径。

## 1. 本地开发模式

### 启动前端

```powershell
cd C:\Users\21963\WeChatProjects\zz\cloud-mp\admin-ui
npm install
npm run dev
```

### 确认管理 API 可访问

前端默认会把请求代理到：

```text
http://127.0.0.1:3001/admin/api
```

如果你的本地 API 端口不同，请设置：

```powershell
$env:VITE_ADMIN_DEV_PROXY_TARGET='http://127.0.0.1:你的端口'
npm run dev
```

## 2. 生产构建

```powershell
cd C:\Users\21963\WeChatProjects\zz\cloud-mp\admin-ui
npm run build
```

当前正式后台入口约定：

- 页面：`https://jxalk.wenlan.store/admin/`
- API：`https://jxalk.wenlan.store/admin/api`

## 3. 登录前提

登录依赖 `cloudfunctions/admin-api` 中已有管理员数据，至少需要：

- `admins`
- `admin_roles`

如果是新环境，应先准备 CloudBase 集合和导入数据，再验证登录。

## 4. 当前页面范围

核心页面包括：

- 看板与运营参数
- 商品、分类、活动、优惠券
- 订单、自提、退款、押金、提现、佣金
- 用户、经销商、分支代理
- 内容、装修、素材、评论、群发
- 会员策略
- 管理员、运维监控、日志

## 5. 常见问题

### 启动后接口 404

- 检查本地 API 是否挂在 `/admin/api`
- 检查 `VITE_ADMIN_DEV_PROXY_TARGET`

### 登录后请求 401

- 检查管理员账号是否存在
- 检查 `admin-api` 的 JWT 配置

### 本地打开构建产物后接口错误

- 静态打开只适合排障
- 正式访问应通过已配置域名和路由的托管环境
