# 问兰 · 微信电商分销系统（仓库 zz）

本仓库实现「问兰」主业务：**微信小程序商城 + 管理后台 + 后端 API**，覆盖订单、支付、分销佣金、代理升级、活动运营、自提等能力。三端目录：

- `backend/`：后端 API、业务服务、定时任务、管理端接口
- `admin-ui/`：管理后台（Vue 3 + Vite + Pinia + Element Plus）
- `miniprogram/`：微信原生小程序

## 业务定位（产品口径摘要）

项目不是单一货架商城，而是 **线上成交 + 线下履约（门店/自提）+ 身份与团队成长 + 资源招商 + 区域经营 + 奖励结算与风控** 的复合体系。当前阶段优先闭环：商品与订单、爆品承接、团队/资源模式基础、自提与区域规则、犹豫期与退款及分佣规则的系统化。

更完整的产品与商业模式说明见下文「文档地图」中的总需求文档；**与代码、接口、字段是否一致，以仓库内架构文档及实现为准**。

## 代理体系 4.0 结构速览（对照脑图）

脑图将业务拆为两条主入口（实现上可能对应小程序消费端与公众号/代理端等，**以 `项目业务总览.md` 与代码为准**）：

- **客户（小商城）｜消费端**：`C0` → `C1（初级代理）` → `C2（高级代理）`，含注册获权、直推分销比例、复购折扣等；脑图备注示例：复购折扣不包含爆单和折扣装等。
- **代理网页（公众号）｜代理端**：`B1（推广合伙人）`、`B2（运营合伙人，3w 档）`、`B3（区域合伙人，19.8w）`，含 40% 分销/成本/利润池拆分、动销协助奖、平级奖、基金池与排行类分配等；并含权益保障、退款范围与周期等对内约定。

代码中除标准 C/B 路径外，另有 **N 路径（小 n / 大 N 独立代理）**，详见 [`docs/architecture/项目业务总览.md`](./docs/architecture/项目业务总览.md) 第三节。

## 开发命令

以下以 macOS / Linux（bash）为例；Windows 可将 `cd` 与 `npm` 命令在 PowerShell 或 CMD 中同等执行。

### Backend

```bash
cd backend
npm install
npm run dev
```

测试：

```bash
cd backend
npm test
```

后端测试与集成验证持续完善中，详见审计与 `docs/audit/` 下相关说明。

### Admin UI

```bash
cd admin-ui
npm install
npm run dev
```

构建：

```bash
cd admin-ui
npm run build
```

### Miniprogram

使用微信开发者工具打开 `miniprogram/`；运行依赖本地配置与后端服务。
